Principles Used:
  • Single Responsibility - Each Product class has its own defined responsibility. ProductsModel defines the data model, ProductsService defines the service to fetch the data, and ProductsController defines the controller to handle the API requests.
  • Open/Closed - The ProductsController class is open for extension but closed for modification. The controller can be extended to handle more API requests, but the existing methods will not be modified.
  • Liskov Substitution - Not Applicable - No subclasses are used in the Product model.
  • Dependency Inversion - The ProductsController class is injected with the ProductsService class to handle the API requests.
  • Interface Segregation - The ProductsController class implements the IProductsController interface, which defines the methods that the controller must implement.
